| Property          | Value                                                                        |
| ----------------- | ---------------------------------------------------------------------------- |

## Spacing Scale

Declared values — sourced from `packages/theme/src/spacing.ts`:

| Token | Value | Usage                                    |
| ----- | ----- | ---------------------------------------- |
| xs    | 4px   | Icon gaps, inline padding                |
| sm    | 8px   | Compact element spacing                  |
| md    | 16px  | Default element spacing, form field gaps |
| lg    | 24px  | Section padding, card inner padding      |
| xl    | 32px  | Layout gaps between major sections       |
| 2xl   | 48px  | Major section breaks                     |

Exceptions: Touch targets use `minHeight: 48, minWidth: 48` (from `touchTarget` token) — trucker-friendly minimum for all interactive elements.

> Note: This phase introduces no new UI screens. The spacing scale is the existing system baseline for downstream phases to consume.

## Typography

Sourced from `packages/theme/src/typography.ts`. All sizes are React Native numeric values (density-independent pixels).

| Role    | Size | Weight         | Line Height | Usage                                     |
| ------- | ---- | -------------- | ----------- | ----------------------------------------- |
| Body    | 16   | 400 (regular)  | 1.5         | Form labels, input text, body copy        |
| Label   | 14   | 400 (regular)  | 1.4         | Helper text, secondary labels, timestamps |
| Heading | 22   | 600 (semibold) | 1.2         | Screen titles, section headings           |
| Display | 28   | 700 (bold)     | 1.1         | App name / brand headline                 |

Weights declared: regular (400) and semibold (600). Bold (700) is reserved for the display role only (existing brand usage in login screen title).

> Minimum body size is 18px per theme comments ("Minimum for Trucker Mode") — the 16px body above is for non-critical secondary copy only. Any primary action copy or input placeholder uses 18px (`fontSize.large` token).

## Color

Sourced from `packages/theme/src/colors.ts` — "Trucker-Friendly High Contrast Theme".

| Role            | Value     | Usage                                                        |
| --------------- | --------- | ------------------------------------------------------------ |
| Dominant (60%)  | `#FFFFFF` | Screen backgrounds, form surfaces, input backgrounds         |
| Secondary (30%) | `#F5F5F5` | Card backgrounds, disabled states, section dividers          |
| Accent (10%)    | `#FF6B35` | Primary CTA buttons, active input borders, loading indicator |
| Destructive     | `#EE5A6F` | Error inline messages, destructive action buttons            |

Accent (`#FF6B35` — high-visibility orange) reserved for: primary submit button fill, active/focused input border color, activity indicator on loading screen.

> Note: The dark background `#1a1a2e` (splash screen, adaptive icon) is a brand color for app launch only — not used in authenticated screens.

## Copywriting Contract

Phase 1 introduces one new user-facing string: the API error response for the `GET /api/auth/mobile-config` endpoint. All other copywriting in this phase is infrastructure (CDK construct names, SSM description strings) which is not user-facing.

| Element                  | Copy                                                                                                                                                             |
| ------------------------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Primary CTA              | not applicable — no new screens in this phase                                                                                                                    |
| Empty state heading      | not applicable — no new screens in this phase                                                                                                                    |
| Empty state body         | not applicable — no new screens in this phase                                                                                                                    |
| Error state (API 400)    | "Tenant not found" — returned as `{ error: "Tenant not found", code: "TENANT_NOT_FOUND" }` when `tenantId` query param does not match a known tenant             |
| Error state (API 500)    | "Authentication service misconfigured" — returned as `{ error: "Authentication service misconfigured", code: "INTERNAL_ERROR" }` when Lambda env vars are absent |
| Destructive confirmation | none — no destructive actions in this phase                                                                                                                      |

> Source: CONTEXT.md — "API handler error message wording for the 400 response" is under Claude's Discretion. Wording above is prescriptive for executor use and follows the existing error shape in `packages/api/src/handlers/auth.ts`.
