## Scope Note

Phase 3 is a service/logic replacement phase. No new screens are introduced. No new visual components are added. The only UI-visible changes are:

1. `settings.tsx` — "DRIVER PROFILE" card drops `driverName`; shows `session?.email` and `session?.role` instead.
2. `_layout.tsx` — The `isLoading` spinner (existing `ActivityIndicator`) covers cold-start restore. Behaviour is unchanged; it already handles `isLoading === true`. No visual change needed.
3. `login.tsx` — No visible change in Phase 3. The screen still renders; it cannot complete a real login (no `tenantId` source yet). The hint text must be updated to remove the demo instruction. Phase 4 owns the two-step flow.

All other UI contract items below specify exact values the executor must match against the established `@pegasus/theme` tokens. Do not deviate.

| Role    | Size | Weight | Line Height | Token     |
| ------- | ---- | ------ | ----------- | --------- |

| Value   | 18px | 600    | 1.4         | large     |

- 18px (`large`) is the minimum for trucker-mode readability — enforced by existing theme comment.

## Interaction Contract

### Logout flow (SESSION-03)

1. Driver taps "LOG OUT" button on settings screen.
2. `Alert.alert('Log Out', 'Are you sure you want to log out?', [...])` fires.
3. On "Log Out" (destructive): `logout()` is called.
4. `logout()` deletes secure-store entry, sets `session` to null, `isAuthenticated` to false.
5. Auth guard in `_layout.tsx` detects `isAuthenticated === false` and redirects to `/(auth)/login`.
6. No intermediate loading state shown during logout.

### Session expiry (SESSION-04)

1. App comes to foreground (`AppState` change event fires).
2. If `session !== null && session.expiresAt < Date.now()`: call `logout()`.
3. `logout()` follows the same path as above — clears store, routes to login.
4. No toast, alert, or modal shown. The login screen itself is the re-prompt.

### Cold-start restore (SESSION-02)

1. App launches. `isLoading` is `true`. Spinner renders over `colors.background`.
2. `checkSession()` reads secure store.
3. If valid session found: set `session`, `isAuthenticated = true`, `isLoading = false`. Auth guard routes to `/(tabs)`.
4. If no session or expired: `session = null`, `isLoading = false`. Auth guard routes to `/(auth)/login`.
5. Driver never sees the login screen if already authenticated.
