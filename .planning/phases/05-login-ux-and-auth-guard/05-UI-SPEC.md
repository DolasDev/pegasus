## Spacing Scale

Declared values from `packages/theme/src/spacing.ts` (all multiples of 4):

| Token | Value | Usage                                                           |
| ----- | ----- | --------------------------------------------------------------- |
| xs    | 4px   | Icon gaps, fine-grained separators                              |
| sm    | 8px   | Label-to-input gap (`marginBottom: spacing.sm` on label)        |
| md    | 16px  | Not currently used in login.tsx; available for internal padding |
| lg    | 24px  | Input container bottom margin; button top margin; input padding |
| xl    | 32px  | Page content padding (`padding: spacing.xl` on content wrapper) |
| xxl   | 48px  | Header bottom margin (`marginBottom: spacing.xxl`)              |

Exceptions:

- Touch targets: `touchTarget.minHeight = 48` and `touchTarget.minWidth = 48` applied to TextInput (`minHeight`) and TouchableOpacity buttons (`minHeight`). This is a trucker-friendly accessibility requirement, not a spacing token override.
- The show/hide toggle sits inside the input container at right-side alignment. No additional spacing token is introduced — it uses `spacing.lg` (16px) for its own horizontal padding, matching the existing input padding.

Source: `packages/theme/src/spacing.ts`

## Color

All values from `packages/theme/src/colors.ts`.

| Role            | Token            | Hex       | Usage                                                                            |
| --------------- | ---------------- | --------- | -------------------------------------------------------------------------------- |
| Dominant (60%)  | `backgroundDark` | `#1a1a2e` | Full-screen container background (`styles.container`)                            |
| Secondary (30%) | `background`     | `#FFFFFF` | TextInput fill (`backgroundColor: colors.background` on `styles.input`)          |
| Accent (10%)    | `primary`        | `#FF6B35` | Submit buttons, subtitle text, company name display, SHOW/HIDE toggle label text |
| Destructive     | `error`          | `#EE5A6F` | Inline error text only — `emailError` (existing), `passwordError` (this phase)   |

Accent (`#FF6B35`) is reserved for:

1. Submit / action button backgrounds (`styles.button`)
2. "Driver Portal" subtitle text (`styles.subtitle`)
3. Company name confirmation text (`styles.companyName`) — password step only
4. SHOW/HIDE toggle label text (this phase — D-05 from CONTEXT.md)

Accent is NOT used for: borders, backgrounds, disabled states, error states.

Disabled state: `colors.textDisabled` (`#9E9E9E`) — used for button background when `isLoading` is true (`styles.buttonDisabled`).

Border color: `colors.border` (`#E0E0E0`) — TextInput border at 2px width.

Primary focal point: the "LOG IN" / "FIND MY COMPANY" submit button via accent (`#FF6B35`) background. Secondary anchor: the app title at 36px display weight 700.

Source: `packages/theme/src/colors.ts`; CONTEXT.md D-05.

## Interaction Contract

### Password Show/Hide Toggle (AUTH-04)

- Toggle is a `TouchableOpacity` inside a flex-row wrapper that replaces the bare `TextInput` for the password field.
- Label text: `"SHOW"` when `secureTextEntry={true}`; `"HIDE"` when `secureTextEntry={false}`.
- Label style: `fontSize.medium` (16px), `fontWeight: '700'`, `color: colors.primary` (`#FF6B35`), `letterSpacing: 0.5` (matches `styles.label`).
- Toggle sits flush to the right inside the input container, vertically centred.
- Toggle is only rendered on the password step, not the email step.
- Source: CONTEXT.md D-04, D-05.

### Inline Error States (AUTH-05)

All errors appear as `Text` below the relevant input using `styles.errorText` (already defined in `login.tsx`):

- `color: colors.error` (`#EE5A6F`)
- `fontSize: fontSize.medium` (16px)
- `marginTop: spacing.sm` (8px)

Error message map (exact copy, from CONTEXT.md D-03):

| AuthError code              | Displayed message                                       |
| --------------------------- | ------------------------------------------------------- |
| `NotAuthorizedException`    | "Incorrect password. Please try again."                 |
| `UserNotFoundException`     | "Account not found."                                    |
| `UserNotConfirmedException` | "Account not confirmed. Contact your company admin."    |
| `LimitExceededException`    | "Too many attempts. Please wait and try again."         |
| Network / unknown           | "Unable to connect. Check your internet and try again." |
| Empty password (local)      | "Please enter your password."                           |

`Alert.alert` is removed from both the empty-password guard and the login failure path.

### Input Locking (AUTH-06)

- `editable={!isLoading}` on all `TextInput` components (email and password steps).
- `disabled={isLoading}` on all `TouchableOpacity` submit buttons.
- `isLoading` is set to `true` before the first async call on each step and set to `false` only after the call resolves or throws.
- SHOW/HIDE toggle remains tappable while `isLoading` is true (it has no async side effect).

### Auth Guard (GUARD-01)

- `SplashScreen.preventAutoHideAsync()` called at module scope in `_layout.tsx` (outside component body).
- `SplashScreen.hideAsync()` called inside `useEffect` when `isLoading` (from `useAuth()`) transitions to `false`.
- `useEffect`-based `router.replace` redirect and `ActivityIndicator` spinner are removed entirely.
- `Stack.Protected` wraps authenticated screens with `guard={isAuthenticated}`.
- `useSegments` and `useRouter` imports in `_layout.tsx` are removed once the guard replaces them.
- Source: CONTEXT.md D-08, D-09, D-10, D-11.
