# Plan 1: Source code fixes — BREAK-01 commit, audience array, env guard, Session type

## Objective

Commit the already-in-place BREAK-01 field name fix, apply the audience array and env guard changes to `validate-token`, and extend the `Session` type with `ssoProvider`. These are all source-only changes with no new tests — the test suite is written in Plan 2 once the types are stable.

## Context

- **BREAK-01** (`authService.ts` line 71): The fix is already in the working tree. `body: JSON.stringify({ token: idToken })` has been changed to `body: JSON.stringify({ idToken })`. The Zod schema on the server requires `{ idToken: string }`; the old payload produced a `VALIDATION_ERROR` 400 before the JWT was ever verified. Task 1 just stages and commits this file.
- **BREAK-02** (`auth.ts` lines 392–408): `jwtVerify` currently passes `audience: tenantClientId` (a single string). Mobile tokens have `aud` set to `COGNITO_MOBILE_CLIENT_ID`, so they fail the audience check and return 401. The fix adds `mobileClientId` to the env guard and changes `audience` to an array. jose v5 `JWTVerifyOptions.audience` accepts `string | string[]`; passing the array makes jose accept a token if its `aud` matches either element.
- **D-02 (empty string risk)**: If `COGNITO_MOBILE_CLIENT_ID` is unset, `?? ''` produces `''`. The env guard must check `!mobileClientId` before constructing the array — the guard fires before `jwtVerify` is called, so the empty string is never passed.
- **Session type** (`types.ts` lines 13–19): The API already returns `ssoProvider` at `auth.ts` line 477. Adding the field to the mobile `Session` type aligns the type with the actual API response. Two test files (`authService.test.ts` line 5–11 and `AuthContext.test.tsx` line 20–26) have `mockSession` objects that will fail TypeScript compilation until `ssoProvider: null` is added.
- **No CDK or AppEnv changes**: `COGNITO_MOBILE_CLIENT_ID` is already in the CDK Lambda env (api-stack.ts line 116) and is read via `process.env[...]` directly — no typed env interface to update.

### Task 2: Add mobileClientId env guard and audience array to validate-token

**File(s):** `packages/api/src/handlers/auth.ts`

**What to do:**

The `validate-token` handler begins at line 382. Make two targeted changes:

**Change 1 — Env guard (lines 392–398).**

Current code:
```typescript
const jwksUrl = process.env['COGNITO_JWKS_URL'] ?? ''
const tenantClientId = process.env['COGNITO_TENANT_CLIENT_ID'] ?? ''

if (!jwksUrl || !tenantClientId) {
  logger.error('validate-token: COGNITO_JWKS_URL or COGNITO_TENANT_CLIENT_ID not set')
  return c.json({ error: 'Authentication service misconfigured', code: 'INTERNAL_ERROR' }, 500)
}
```

Replace with (add the `mobileClientId` line after `tenantClientId`, extend the guard condition and log message):
```typescript
const jwksUrl = process.env['COGNITO_JWKS_URL'] ?? ''
const tenantClientId = process.env['COGNITO_TENANT_CLIENT_ID'] ?? ''
const mobileClientId = process.env['COGNITO_MOBILE_CLIENT_ID'] ?? ''

if (!jwksUrl || !tenantClientId || !mobileClientId) {
  logger.error(
    'validate-token: COGNITO_JWKS_URL, COGNITO_TENANT_CLIENT_ID, or COGNITO_MOBILE_CLIENT_ID not set',
  )
  return c.json({ error: 'Authentication service misconfigured', code: 'INTERNAL_ERROR' }, 500)
}
```

**Change 2 — audience option in jwtVerify call (line 407).**

Current code:
```typescript
audience: tenantClientId,
```

Replace with:
```typescript
audience: [tenantClientId, mobileClientId],
```

No other lines in the file need changing. The `mobileClientId` variable is already in scope from Change 1 when Change 2 executes.

After editing, commit:
```bash
git add packages/api/src/handlers/auth.ts
git commit -m "fix(api): accept mobile Cognito client ID in validate-token audience (BREAK-02)

Add COGNITO_MOBILE_CLIENT_ID to env guard and pass audience array
[tenantClientId, mobileClientId] to jwtVerify so mobile tokens are
accepted alongside web tokens. Per D-01 and D-02.

Co-Authored-By: Claude Sonnet 4.6 <noreply@anthropic.com>"
```
