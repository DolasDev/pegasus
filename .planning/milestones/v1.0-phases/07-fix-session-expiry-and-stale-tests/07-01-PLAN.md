Fix two independent cross-phase defects confirmed by the v1.0 milestone audit:

1. BREAK-03 / SESSION-04: AuthContext.tsx:56 compares session.expiresAt (JWT seconds, ~1.7e9) against Date.now() (milliseconds, ~1.7e12). Real sessions are always less than Date.now() — every driver is immediately force-logged-out on the first app resume after the initial login. Fix by multiplying expiresAt by 1000 at the comparison site (Option B — no API change, no web package impact). Update all 5 AuthContext.test.tsx fixtures from millisecond-scale to seconds-scale to match.

The test type-asserts { token: string } and checks body.token, which is now undefined. Fix by updating the type assertion to { idToken: string } and checking body.idToken.

Purpose: SESSION-04 is the last unsatisfied requirement blocking the v1.0 milestone. MISSING-01 is a test defect masking the BREAK-01 fix.
