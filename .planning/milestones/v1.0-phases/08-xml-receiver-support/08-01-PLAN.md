<objective>
Add XML event_data support to the Pegasus Events Receiver.

The API response remains JSON throughout — the outer envelope, event metadata
(event_id, event_type), and list structure are unchanged. The only difference is
that when XML format is selected, the value at event['event_data']['S'] is a
serialized XML string rather than a nested JSON dict.

The service must parse that XML string into the same
`{'data': [{'name': '<TABLE>', 'values': {...}}, ...]}` shape that insertIntoDB()
already expects, then continue processing identically.

Output: one new module, four modified files, zero changes downstream of
insertIntoDB().
</objective>

## Task 3 — New module: XML event_data parser

**File**: `apps/services/service/app/xml_handler.py` (create new)

Parses a serialized XML string from `event['event_data']['S']` and returns the
`{'data': [...]}` dict that `insertIntoDB()` expects.

```python
import xmltodict
from app.loggers import logger


def parseEventDataXml(xml_string: str) -> dict:
    """
    Convert a serialized XML event_data string to the dict shape expected by
    ControlFlow.insertIntoDB():

        {'data': [{'name': '<TABLE_NAME>', 'values': {<field>: <value>, ...}}, ...]}

    Expected XML structure:
        <EventData>
          <Table name="LEAD">
            <ID>1</ID>
            <FIRST_NAME>Jane</FIRST_NAME>
          </Table>
          <Table name="ADDRESS">
            ...
          </Table>
        </EventData>
    """
    logger.info('Running parseEventDataXml()')
    parsed = xmltodict.parse(xml_string)

    root_key = list(parsed.keys())[0]   # e.g. 'EventData'
    tables_raw = parsed[root_key].get('Table', [])

    # xmltodict wraps a single child element as a dict, not a list
    if isinstance(tables_raw, dict):
        tables_raw = [tables_raw]

    data = []
    for table in tables_raw:
        table_name = table.pop('@name', table.pop('name', 'UNKNOWN'))
        # Remaining keys are field-value pairs; normalize None → ''
        values = {k: (v if v is not None else '') for k, v in table.items()}
        data.append({'name': table_name, 'values': values})

    return {'data': data}
```

### 4b — Add normalisation helper

Add this function anywhere before `insertIntoDB()`:

```python
def normaliseEventData(event):
    """
    If the service is configured for XML input, parse the serialized XML string
    at event['event_data']['S'] into the standard dict shape.
    Returns the event dict with event_data['S'] replaced by the parsed dict.
    Mutates and returns the same dict for simplicity.
    """
    if config.input_format == 'xml':
        xml_string = event['event_data']['S']
        event['event_data']['S'] = xml_handler.parseEventDataXml(xml_string)
    return event
```
