<objective>
Install `react-native-get-random-values` and `amazon-cognito-identity-js` via `npx expo install` (Expo-managed version resolution), then prepend `import 'react-native-get-random-values'` as the absolute first line of `apps/mobile/app/_layout.tsx`.

Purpose: INFRA-01. The Cognito SRP handshake (`amazon-cognito-identity-js`) calls `crypto.getRandomValues()` during authentication. React Native's bundled JS engine does not include a WebCrypto implementation. The polyfill must be loaded before any module that touches crypto — meaning it must be the first import in the module that is loaded first by expo-router, which is `app/_layout.tsx`.

Why NOT apps/mobile/index.ts: `apps/mobile/package.json` sets `"main": "expo-router/entry"`. Metro resolves `expo-router/entry` → `entry-classic.js` → the expo-router file system, which loads `app/_layout.tsx`. The `index.ts` file is bypassed and is NOT part of the bundle graph. Placing the polyfill there would have no effect. D-08 refers to the logical "app entry" — `_layout.tsx` achieves the same runtime guarantee via the correct mechanism for expo-router projects.

Output: Two packages installed in apps/mobile/package.json, polyfill as line 1 of \_layout.tsx.
</objective>
