## Tech Debt by Phase

No critical blockers. All tech debt is operational or documentation in nature.

### Phase 04: Tenant Resolution Flow

- **providers field superset (info):** API resolve-tenants/select-tenant responses include a `providers` array not declared in mobile `TenantResolution` type. Mobile casts and silently discards it. Not a runtime break; documents a divergence between API response and mobile type contract.
- **TENANT-06 device test pending:** Back navigation is structurally correct (`router.push` + `headerShown: true`) but requires physical device/simulator to confirm Expo Router native stack-pop behavior.

### Phase 02: Auth Service Layer

- **Missing .env.example:** `apps/mobile/` has no `.env.example` documenting `EXPO_PUBLIC_API_URL`. `_layout.tsx` falls back to `''`; all `fetch` calls silently fail on physical device builds without this variable set.

### Phase 01: Infrastructure Foundation

- **CDK deploy not live-confirmed:** 117 CDK tests pass against synthesized CloudFormation template. Actual AWS mobile app client provisioning and Lambda env var injection require a live deployment to confirm.
- **Device test pending:** Cognito SRP polyfill Metro bundle execution order requires running device/simulator to verify.

### Phase 05: Login UX and Auth Guard

- **Three device tests pending:** SplashScreen cold-start flash elimination, SHOW/HIDE toggle text masking, and `editable={false}` input locking are all native OS behaviors unverifiable by Jest.

### Phase 06: Fix Mobile Token Validation

- **Two deployment tests pending:** End-to-end mobile auth on real device and `COGNITO_MOBILE_CLIENT_ID` Lambda env var deployment confirmation require live environment.

**Total tech debt items:** 9 across 5 phases. All are operational/device-verification items or documentation gaps. No code defects.
