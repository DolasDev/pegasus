# tmux config — decisions & usage

Optimised for Termius mobile access. Each section documents what changed from
the tmux default and how to use it.

---

## Prefix: `C-a` (default: `C-b`)

The default `C-b` requires stretching your left hand awkwardly, especially on
a phone keyboard. `C-a` sits on the home row and is the classic screen prefix.

**Usage:** press `Ctrl+A`, release, then press the next key.

---

## Mouse: on (default: off)

Termius translates touch gestures into mouse events. With mouse enabled:

- **Tap a pane** to focus it
- **Tap a window tab** in the status bar to switch to it
- **Scroll** (swipe up/down) to move through scrollback history
- **Drag a pane border** to resize it

No prefix needed for any of the above.

---

## `escape-time 20` (default: 500 ms)

tmux waits up to 500 ms after `Escape` to decide if it is the start of a key
sequence. Over a mobile network this causes noticeable lag every time you press
`Escape` (e.g. in vim). Reduced to 20 ms.

No usage change — just faster response to `Escape`.

---

## `repeat-time 300` (default: 500 ms but only for `-r` bindings)

Applies to repeatable bindings (resize keys). 300 ms gives a slightly longer
window to press the key again without re-issuing the prefix — helpful when
touch-typing is less precise.

**Usage:** `C-a H H H` resizes the pane left by 15 columns (3 × 5) without
needing to re-press `C-a` between each `H`, as long as each keystroke arrives
within 300 ms.

---

## `history-limit 50000` (default: 2000)

More scrollback is useful on a phone where you cannot see much at once and
often need to scroll back to review command output.

**Usage:** enter copy mode with `C-a [`, then scroll with arrow keys or swipe.
`q` exits copy mode.

---

## `base-index 1` / `pane-base-index 1` (default: 0)

Windows and panes are numbered from 1 instead of 0. The `1` key on a phone
keyboard maps directly to window 1, avoiding the awkward zero.

**Usage:** `C-a 1` through `C-a 5` jump directly to those windows.

---

## `renumber-windows on` (default: off)

When you close a window, remaining windows are renumbered to stay sequential.
Prevents gaps like `1, 3, 5` after closing windows mid-list.

No usage change — window numbers just stay tidy automatically.

---

## Split bindings: `|` and `-` (default: `%` and `"`)

The defaults (`%` and `"`) are harder to remember and require `Shift` on most
keyboards. Pipe and hyphen are mnemonic: `|` = vertical line = vertical split
(side by side), `-` = horizontal line = horizontal split (stacked).

**Usage:**

- `C-a |` — split current pane left/right
- `C-a -` — split current pane top/bottom
- New panes open in the same directory as the current pane.

---

## Pane navigation: arrow keys (replaces `h/j/k/l` default)

The default vim-style navigation (`C-a h/j/k/l`) conflicts with terminal
applications and requires remembering which direction each letter maps to.
Arrow keys are unambiguous and easier to tap on a phone.

**Usage:** `C-a ↑ / ↓ / ← / →` to move between panes.

---

## Pane resize: `H J K L` with `-r` repeat (default: same keys but smaller step)

Resize by 5 cells per keypress (default is 1). Coarser steps suit imprecise
touch input. The `-r` flag means you can hold the key sequence to keep resizing.

**Usage:** `C-a H` (shift+h) shrinks the pane left 5 columns. Hold or repeat
within the `repeat-time` window for larger adjustments.

---

## `new-window` preserves path: `-c "#{pane_current_path}"` (default: home dir)

New windows and splits open in the same directory as the active pane, not `~`.
Eliminates the need to `cd` after every split when working deep in a project.

Applies to `C-a c`, `C-a |`, and `C-a -`.

---

## Copy mode: vi keys (default: emacs)

`set -g mode-keys vi` switches copy mode to vi bindings, which are more
widely known in a developer context.

**Usage:**

1. `C-a [` — enter copy mode
2. Navigate with arrow keys or `h/j/k/l`
3. `v` — start selection
4. `y` — yank (copy) selection and exit copy mode
5. `C-a ]` — paste

---

## Reload binding: `C-a r` (not in default config)

Sources `~/.tmux.conf` without restarting tmux. Displays a confirmation message.

**Usage:** `C-a r` after editing this config to apply changes immediately.

---

## Kill confirmations: `C-a q` / `C-a Q` (not in default config)

Prompts before killing a pane (`q`) or window (`Q`) to prevent accidental
closes — especially useful with imprecise touch input.

**Usage:** `C-a q` → type `y` to confirm pane kill, `n` to cancel.

---

## Activity monitoring (default: off)

`monitor-activity on` marks windows with pending output in the status bar
(highlighted in amber). Useful when you kick off a long-running command, switch
to another window, and want to know when it finishes.

`visual-activity off` suppresses the bell/flash — the status bar indicator is
enough.

No action needed — watch the window list in the status bar.
