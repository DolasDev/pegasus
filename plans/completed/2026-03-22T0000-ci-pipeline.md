# GitHub Actions CI Pipeline

**Branch:** `feature/on-prem-server`
**Goal:** Add a GitHub Actions CI pipeline with typecheck, lint, test, and E2E gates so no broken code can be merged to main.

## Context

No CI/CD exists. Developers can merge broken tests, type errors, or lint failures. This is the single highest-leverage improvement.

## Implementation Checklist

### 1. CI workflow file

- [x] Write test: manually verify workflow syntax with `actionlint` or dry-run
- [x] Create `.github/workflows/ci.yml`
  - Trigger on `push` to main, `pull_request` to main
  - Jobs: `typecheck`, `lint`, `test` (with Postgres service), `e2e` (with Postgres + Playwright)
  - Postgres 16 service container with credentials matching `docker-compose.yml`
  - `chmod +x` step for WSL2 binary permissions quirk
  - `npx prisma migrate deploy` before integration tests
  - `npx playwright install --with-deps chromium` in E2E job
  - Upload `playwright-report/` as artifact on failure
  - `concurrency` group to cancel stale PR runs

### 2. Verify

- [x] Push branch, open PR, confirm all checks trigger
- [x] Confirm merge is blocked on check failure

## Risks / Side Effects

- CI secrets (DATABASE_URL etc.) must be configured in GitHub repo settings
- Playwright browser install adds ~2 min to E2E job
- WSL2 chmod step is a workaround; ideally fixed upstream

## Dependencies

None — can start immediately.
